웹 경험 미리보기 및 게시

이 항목에서는 웹 경험을 미리보고 게시하는 방법을 배웁니다.

웹 경험은 개발 중에 쉽게 미리 볼 수 있습니다. 경험을 미리 볼 수있는 링크도 공유 할 수 있습니다. 경험이 완료되면 공개적으로 사용할 수 있도록 게시 할 수 있습니다.

웹 경험 미리보기

웹 경험 미리보기를 통해 모바일, 태블릿 및 데스크탑 장치에서 경험이 어떻게 표시되는지 확인할 수 있습니다. 웹 경험을 미리 보려면 경험을 편집 한 다음시사경험 편집기에서.

두 가지 옵션이 있습니다.

  • 반응형 미리 보기 - 미리 보기를 위해 새 브라우저 창에서 환경을 엽니다.
  • 공유 가능한 링크 가져오기 - 미리 보기를 다른 사용자와 공유할 수 있도록 미리 보기 URL을 제공합니다.

생성된 공유 가능 URL은 72시간 동안 유효합니다.

참고: 액세스 제어 설정의 일부로 세션 시간 초과를 구성한 경우 72시간 제한 시간이 구성된 세션 시간 초과를 무시합니다. 경험이 게시되면 세션 시간 초과 (있는 경우) 가 적용됩니다.

웹 경험 게시

Web Experience를 게시하면 뷰어가이를 사용할 수 있으며 사이트에 대한 정적 URL이 생성됩니다. 경험을 게시하려면 다음 단계를 따르십시오.

참고: 최근에 비디오 메타데이터 또는 재생 목록을 변경한 경우 이러한 변경 사항이 게시된 경험에 표시되는 데 몇 분 정도 걸릴 수 있습니다.
  1. 경험 이름을 클릭하여 경험 편집기에서 사이트를 엽니 다.
  2. 게시를 클릭합니다 .
  3. 게시 확인 대화 상자에서 지금게시를 클릭합니다 .
  4. 경험이 게시되고 사이트 URL이 표시됩니다.

bcvp0rtal도메인은 기본적으로 SSL을 지원하므로 https: //를 사용하여 사이트에 액세스할 수도 있습니다. 설정이나 SSL 인증서가 필요하지 않습니다.

경험이 게시되면 URL이 사이트 세부 정보의 일부로 표시됩니다. 프로젝트홈페이지.

웹 경험에 변경 사항 게시

경험을 변경할 때는 다시 게시해야 합니다. 다시 게시해야 하는 경험에는 게시되지않은 변경 사항의 경험 상태가 표시됩니다 .

변경 사항을 다시 게시하려면에 설명 된 단계를 따르십시오. 웹 경험 게시이 항목의 섹션.

참고: 변경 사항이 나타나지 않으면 브라우저 캐시와 쿠키를 지워야 할 수 있습니다.

웹 경험을 오프라인으로 전환

경험을 오프라인으로 전환하면 해당 URL에서 더 이상 사용할 수 없습니다. 경험을 오프라인으로 전환하려면 행을 클릭한 다음오프라인으로 전환을 클릭합니다 . 경험을 오프라인으로 전환해야 하는지 확인하려면 오프라인으로 전환을 클릭합니다 . 경험을 다시 사용할 수 있도록 하려면 경험을 다시 게시해야 합니다. 경험이 오프라인 상태가되면 더 이상 해당 URL을 통해 연결할 수 없습니다.

참고: 경험을 오프라인으로 전환해도 경험은 삭제되지 않습니다.
