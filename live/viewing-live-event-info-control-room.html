관제실을 사용하여 라이브 이벤트 정보보기

이 항목에서는 Control Room을 사용하여 라이브 이벤트 정보를 보는 방법을 배웁니다.

Control Room 페이지는 라이브 이벤트에 대한 정보를 보는 데 사용됩니다. 새 라이브 이벤트를 만들고 스트리밍이 시작되면 이벤트 정보를 표시하는 Control Room 페이지가 열립니다. 이벤트 미리보기가 미리보기 플레이어에 표시됩니다. 제어실 페이지는 라이브 이벤트 관리 페이지에서 이벤트를 클릭 할 때도 표시됩니다.

제어실 페이지이벤트 정보탭은 다음 섹션으로 구성됩니다.

  • 이벤트 정보
  • 스트림 정보
  • 클립
  • 감시 장치

이벤트 정보보기

그만큼이벤트 정보섹션에는 다음 정보가 표시됩니다.

  • 이벤트 명 -행사 명
  • 간단한 설명 -이벤트에 대한 간단한 설명
  • 태그 -이벤트와 관련된 태그입니다. 태그는 검색을 용이하게하며 스마트 재생 목록을 만들 때 사용할 수 있습니다.
  • 참조 ID -이벤트에 대한 내부 참조 ID
  • 소셜 미디어로 스트리밍 - 허용되는경우 예정된 소셜 이벤트에 라이브 이벤트를 할당할 수 있습니다.
  • 디지털 권한 관리 (DRM) -
  • 재생 URL -비 Brightcove Player에서 라이브 스트림을 보는 데 사용할 수있는 URL

스트림 정보보기

그만큼스트림 정보섹션에는 다음 정보가 표시됩니다.

  • 작업 ID -내부 작업 ID
  • 비디오 ID -라이브 이벤트 스트림의 동영상 ID (동영상 페이지의 VOD 자산에 대한 동영상 ID이기도 함)
  • 끝점 (RTMP URL )-인코더에서 사용하는 스트리밍 엔드 포인트
  • 이름 -인코더에서 사용하는 스트림 이름 (항상살아 있는 )
  • 부위 -스트리밍 지역
  • 상태 -현재 작업 상태
    • 스트리밍 - 스트림이 라이브 중입니다.
    • 처리 오류 - 라이브 스트림에 오류가 발생했습니다.
    • 대기 - 스트림 활성화 준비(정적 진입점만 해당)
    • 멎는 - 라이브 스트림이 중지되고 관련 VOD 출력이 생성됩니다.
    • 완전한 - 라이브 스트리밍이 완료되었습니다.
    • 취소 - 라이브 스트림이 중지되고 보류 중인 VOD 출력을 처리하지 않습니다.
    • 취소 된 - 라이브 스트리밍이 취소되었습니다.
    • 연결 끊김 - 인코더가 분리되었습니다.
    • 대기 중 - RTMP 스트림 대기(인코더 시작)
    • 실패한 - 시스템 오류로 인해 작업이 중지되었습니다.

클립보기

그만큼클립섹션에는 라이브 스트림에서 생성 된 (있는 경우) 모든 클립이 표시됩니다. 클립 이름을 클릭하면동영상페이지. 클립 만들기에 대한 자세한 내용은 비디오 클립만들기를 참조하십시오 .

스트림 모니터링

그만큼감시 장치섹션은 출력 및 입력 드리프트 그래프를 표시합니다. 클릭업데이트 된 데이터 및 그래프 검색정보를 보려면 링크를 클릭하십시오. 출력 드리프트는 스트림의 건전성을 측정합니다. 출력 드리프트는 벽시계와 스트림 타임 스탬프 (TS)의 차이로 정의됩니다. 라이브 스트리밍에서 이상적으로 벽시계는 스트림 TS와 동일한 속도로 실행되어야합니다. 스트림 TS가 실시간보다 느리게 도착하면 시스템에 문제 (병목 현상)가 있음을 의미합니다.

입력 드리프트는 입력 스트림의 상태를 측정하며 입력 타임 스탬프 및 실시간 시계와의 차이로 정의됩니다.
