라이브 이벤트 생성 및 관리

이 항목에서는 라이브 이벤트를 만들고 관리하는 방법을 배웁니다.

소개

Brightcove Engage의 라이브 비디오 기능™모바일 앱과 웹 경험 모두에 라이브 이벤트를 브로드캐스트하는 데 사용할 수 있습니다. 라이브 이벤트를 생성하면 현장 인코더에 연결하는 일련의 인코더 설정이 제공됩니다. 모든 트랜스 코딩은 클라우드에서 수행되므로 하드웨어 기반 트랜스 코더가 필요하지 않습니다. 그런 다음 모바일 앱 및 웹 경험을 만들 때 라이브 비디오를 포함 할 수 있습니다.

전제 조건

라이브 이벤트를 방송하려면 다음 항목이 필요합니다.

  • 이벤트 방송용 카메라
  • 자체 인코딩 소프트웨어가 있어야 합니다 (예: Telestream 와이어캐스트또는오픈 브로드캐스터 소프트웨어)

제한 사항

  • 생방송은 20 ~ 30 초 지연됩니다.
  • 스트리밍은 이벤트 시작 후 30 분 이내에 시작해야합니다.
  • Internet Explorer를 사용하는 사용자의 경우 버전 10 이상이 필요합니다.

라이브 이벤트 관리

라이브 이벤트를 시작하려면브라이트코브 인게이지™클릭살다탐색 헤더에서. 이벤트 관리 페이지에는 두 개의 기본 탭이 표시됩니다.

  • 라이브 -진행중인 라이브 이벤트 목록
  • 완료 -완료된 이벤트 목록

표시드롭다운은완료됨탭에 나타나는 이벤트를 필터링하는 데 사용할 수 있습니다.

페이지 본문에는 관련 정보와 함께 라이브 이벤트 목록이 표시됩니다.

  • 작업 상태-라이브 스트림의 상태
    • 가공 -스트림이 라이브입니다
    • 처리 오류 -실시간 스트림에 오류가 발생했습니다.
    • 대기 -활성화 할 스트림 준비 (정적 진입 점 만 해당)
    • 멎는 -라이브 스트림이 중지되고 관련 VOD 출력이 생성됩니다.
    • 완료 -라이브 스트리밍이 완료되었습니다
    • 취소 -라이브 스트림이 중지되고 보류중인 VOD 출력을 처리하지 않습니다.
    • 취소 된 -라이브 스트림이 취소되었습니다
    • 연결 끊김 -인코더 연결이 끊어졌습니다.
    • 기다리는 -RTMP 스트림 대기 (인코더 시작)
    • 실패한 -시스템 오류로 인해 작업이 중지되었습니다.
    • 포기 됨 -엔코더 연결없이 이벤트 종료
  • 완료된 날짜 및 시간
  • 이벤트 이름 및 ID
  • 총 조회수
  • 지속 시간
  • 생성 된 날짜 및 시간 (반복 이벤트에만 해당)

라이브 이벤트 비디오 속성을 보려면 이벤트 이름을 클릭하여 Control Rood를 엽니다 .

새 이벤트 만들기

새 라이브 이벤트를 만들려면 다음 단계를 따르세요.

  1. 이벤트 관리 페이지에서 이벤트생성을 클릭합니다 . NS 라이브 이벤트 만들기페이지가 열립니다.
  2. 이벤트 세부 정보를 입력합니다.
    • 이벤트 명 -이벤트 이름 이 이름은 비디오가 비디오 자산 (최대 255 자)으로 저장된 경우 비디오 페이지에도 나타납니다.
    • 간단한 설명 - 이벤트 설명(최대 250자)
    • 태그 -이벤트와 관련된 태그 비디오가 비디오 자산으로 저장되면 비디오에 태그가 할당됩니다 (최대 128 자 및 1200 태그).
    • 참조 ID - 이벤트에 대한 참조 ID 고유해야 합니다(최대 150자).
  3. 고르다완료되면 이벤트를 비디오 자산으로 변환라이브 비디오의 주문형 버전을 계정에 저장하려면 라이브 이벤트가 종료되는 즉시 VOD 버전을 사용할 수 없으며 업로드 처리가 완료되면 동영상을 볼 수 있습니다.
  4. 에서스트림 옵션섹션에서부위 . 그만큼부위사용할 Amazon Web Services (AWS) 리전입니다. 지연 시간을 줄이려면 인코더가있는 위치에서 가장 가까운 지역을 선택해야합니다. 기본적으로 지역은 다른 대륙에서 스트리밍하는 경우 나쁜 결과를 생성하는 오레곤으로 설정됩니다. 다음 지역이 지원됩니다.
    • 오리건 (기본값)
    • 여자 이름
    • 도쿄
    • 싱가포르
    • 시드니
    • 뭄바이
    • 프랑크푸르트
    • 아일랜드
  5. 고급 옵션을클릭하고 고급 옵션을구성합니다 .
  6. 딸깍 하는 소리이벤트 생성라이브 이벤트를 시작합니다. NS 제어실 페이지다음을 포함한 라이브 이벤트에 대한 세부 정보와 함께 열립니다. 스트리밍 엔드포인트(RTMP URL)그리고스트림 이름인코더에서 사용됩니다.
    참고: 스트림을 처리하는 데 약 1분 정도 소요됩니다.

    기본적으로 인코더는 새 이벤트가 생성될 때부터 30분 이내에 시작해야 합니다.

스트리밍중지버튼을 클릭하여 이벤트를 종료합니다.

참고: 실시간 스트림이 완료되면 인코더가 닫히거나 중지되었는지 확인합니다. 어떤 인코더와 브라이트코브 간에 연결을 열어 두면 데이터가 전송되지 않더라도 예기치 않은 연결 요금이 발생할 수 있습니다.

고급 옵션 구성

새 이벤트를 만들 때 고급 옵션을 구성 할 수 있습니다. 고급 옵션을 구성하려면 이벤트를 만들 때 ADVANCED OPTIONS링크를 클릭합니다.

다음 옵션을 구성 할 수 있습니다.

  • 콘텐츠 암호화 - 라이브 스트림에 적용할 암호화 유형을 선택합니다.
    • 없음 -암호화가 적용되지 않음
    • AES-128 - AES-128 암호화를 사용하여 스트림을 보호합니다. 이 옵션을 선택하면암호화 키를 입력해야 합니다 . 키는 32개의 16진수여야 합니다. 최상의 결과를 얻으려면키생성기를 사용하여128-bit및Hex옵션을 선택하십시오.
      암호화된 라이브 스트림에서는 클립을 만들 수 없습니다.
  • 입력 형식 -Brightcove Live로 전송 될 인코더의 입력 형식입니다. 이 기능은 지금과 같은 라이브 스트림에서 추가 메타 데이터를 지원하며 자막 및 캡션, 오디오 트랙에서 더 많은 유연성을 제공하고 표준 방송 수준 데이터로 간소화 된 디지털 경험을 가능하게합니다. 다음 입력 형식이 지원됩니다.
    • RTMP (기본값)
    • RTP (Real-Time Transport Protocol)
    • SRT (Secure Reliable Transport)
  • RTMP IP 허용 목록 -RTMP 끝점에 연결할 수있는 IP 주소 집합을 입력합니다. RTMP 엔드 포인트에 연결할 수있는 사용자를 제어합니다. 이 옵션은 입력 형식이 RTMP 인 경우에만 나타납니다.
  • CIDR 화이트리스트  -RTP 또는 SRT 끝점에 연결할 수있는 IP 주소 집합을 입력합니다. 이는 RTP 또는 SRT 엔드 포인트에 연결할 수있는 사용자를 제어합니다. 참고 이 옵션은입력 형식이 RTP 또는 SRT인 경우에만 나타납니다.
  • + 다중 CDN 구성 추가 - 추가 CDN의레이블및 URL을입력합니다. 동영상 세그먼트에 대한 기본 CDN을 요청할 수있는 CDN의 화이트리스트입니다. 보조 CDN을 올바르게 구성해야합니다.

관제실을 사용하여 라이브 이벤트 정보보기

Control Room 페이지는 라이브 이벤트에 대한 정보를 보는 데 사용됩니다. 새 라이브 이벤트를 만들고 스트리밍이 시작되면 이벤트 정보를 표시하는 Control Room 페이지가 열립니다. 제어실 사용에 대한 정보는 다음을 참조하십시오. 관제실을 사용하여 라이브 이벤트 정보보기 .

이벤트 복제

이벤트를 복제하면 이전 라이브 이벤트를 만드는 데 사용 된 것과 동일한 설정으로 다른 이벤트를 쉽게 만들 수 있습니다. 이벤트를 복제하려면 다음 단계를 따르십시오.

  1. 이벤트를 클릭하여 Control Room 페이지를 엽니 다.
  2. 딸깍 하는 소리중복 이벤트페이지 상단에 있습니다. 이전 라이브 이벤트의 설정을 사용하여 새로운 라이브 이벤트가 생성됩니다. 필요에 따라 이벤트 속성을 변경합니다.
  3. 이벤트생성을 클릭합니다 .

클립 만들기

이벤트가 진행 중이거나 이벤트가 종료 된 후에 클립을 만들 수 있습니다. 클립을 만들 수 없는 경우 마우스를 가져가면 메시지가 나타납니다. 클립 만들기왼쪽 탐색에서 클립 만들기에 대한 자세한 내용은 비디오 클립만들기를 참조하십시오 .

참고: 라이브 이벤트가 종료된 후 7일 동안 스트림을 클리핑할 수 있습니다.

실시간 스트림 문제 해결

라이브 이벤트를 진행하는 데 도움이 필요하면 Google에문의하십시오 . 가능한 가장 빠른 응답을받을 수 있도록 다음은 문제 해결에 필요한 지원 목록입니다.

  • 스트림에 나타나는 특정 증상입니다. 예를 들어, 전혀 재생되지 않거나 끊기거나 멈 춥니 다.
  • 이 스트림이 과거에 올바르게 작동했는지 여부
  • 인코더에서 사용중인 진입 점 URL
  • 인코딩 소프트웨어 및 하드웨어를 사용하고 있습니다.
  • Brightcove Engage에서 라이브 자산의 비디오 ID ™
  • 인코더에서 게시 지점 호스트로의 추적 경로 결과
